## Технический стоп-лист перед записью

- [ ] Закрыть лишние вкладки браузера и приложения
- [ ] Выключить уведомления (macOS: Фокусировка → Не беспокоить)
- [ ] Разрешение экрана — минимум 1920×1080
- [ ] Терминал: шрифт 14–16pt, тёмная тема, максимально развёрнут
- [ ] Браузер: масштаб 100%, тёмная тема (опционально)
- [ ] Запустить `docker compose up -d` в обоих директориях заранее — не тратьте время в записи на ожидание билда
- [ ] Проверить `curl http://localhost:8001/api/part/` и `curl http://localhost:8000/api/part/` — оба отвечают
- [ ] Открыть заранее: VS Code с проектом, терминал, браузер со Swagger UI

**Инструмент записи**: OBS Studio / Loom / QuickTime (Mac) — любой с захватом экрана + микрофон.

### Блок 4 — Демо: API тесты (03:30–05:30)

**Что показывать на экране**: терминал.

**Что сделать**:
```bash
cd project/automation/api

# Убедиться что сервер живёт
curl http://localhost:8001/api/part/

# Запустить smoke-тесты (быстро, ~15 сек)
pytest tests/ -m smoke -v

# Затем полный прогон или конкретный файл
pytest tests/test_bom.py -v

# Показать итоговую строку: 181 passed
pytest tests/ --tb=no -q
```

> **Совет**: запустите `pytest tests/ -v` заранее и держите вывод под рукой — если прогон идёт больше 2 минут, используйте `--tb=no -q` для быстрого финального снимка.

**Текст:**
> Запускаем. Сначала smoke — критический путь.
> Все зелёные.
> Теперь полный suite — 181 тест.
> Видим: PartCategory, Part CRUD, Revisions, BOM с циклами, StockItem, Auth, Pagination, Error Format.
> Итог: 181 passed, 0 failed. Время — около 30 секунд.

## Советы по монтажу

| Ситуация | Что делать |
|----------|------------|
| Прогон pytest занимает больше 2 мин | Покажите начало (первые 20 тестов), затем вырежьте до финальной строки `181 passed` |
| Playwright тест падает в прямом эфире | Заранее запишите удачный прогон и используйте его; упомяните устно, что некоторые тесты нестабильны из-за InvenTree SPA sidebar |
| Ошибка Docker при записи | Держите `docker ps` открытым рядом; заранее убедитесь что оба контейнера `Up` |
| Нет времени на весь текст | Блоки 2–3 можно объединить в один 1.5-минутный блок "show code" |
